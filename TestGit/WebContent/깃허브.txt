Git & GitHub

-> 소스 형상관리(버전관리)
-> 프로그램에 관련된 모든 정보를 저장하고 관리
=> 대표적인 형상관리 프로그램(개인, 기업)

* 분산 버전관리
  
  서버			-사용자		-사용자
  정보1 ->(1-1)	 정보1-1		정보1
  정보2		 	 정보2		정보2
  
  내컴퓨터		(스테이징영역)		Git 서버
  (이클립스)
  파일1(커밋) -> 파일1(준비) -> 파일1(완료)
  		commit			push
  
* 용어
	pull - git 저장소 서버에서 내컴퓨터의 저장소로 내려받는 것
	commit - 추가, 수정, 삭제 변경사항을 1개의 버전으로 기록
	push - 로컬의 정보를 git 저장소 서버로 올리는 것
	
	* 가능하면 서버에 올릴때(commit & push) 한동작 단위로 처리
	Branch - 서버에 올라오는 버전들을 브런치(기본 master)
		   - 내컴퓨터 브런치 - 로컬브런치, 서버 브런치 - 리모트 브런치
	tag - 특정 버전을 찾기 쉽게 이름을 지정
	Merge - 병합, 여러명이 작업한 코드를 합치기
